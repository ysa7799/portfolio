# Data Quality Checklist Template

A practical, reusable checklist for validating any dataset (schema, completeness, duplicates, outliers, consistency) before it's used in analysis or a dashboard, paired with an issue-tracking template.

## Files
- `checklist.md` — the validation checklist itself, organized by category
- `issue_log_template.csv` — a sample issue log showing how findings get tracked, prioritized (severity), and closed out

## Why this matters
Most data quality problems aren't algorithmically hard to catch — they're missed because no one checked systematically. This template turns "eyeball the data" into a repeatable process with a paper trail, which matters both for catching recurring errors and for building stakeholder trust in reported numbers.

## Skills demonstrated
Data validation methodology, documentation discipline, issue tracking and severity triage — the same discipline behind auditable incident logging, applied to data pipelines.
